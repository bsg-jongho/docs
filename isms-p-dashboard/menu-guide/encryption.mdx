<Warning>
  암호화는 데이터 유출 시 최후의 방어선입니다. **민감 데이터**(개인정보, 인증정보, 금융정보 등)가 포함된 리소스는 반드시 암호화되어야 합니다.
</Warning>

## 핵심 KPI

### 전체 암호화 준수율

이 수치는 전체 리소스 대비 암호화된 리소스의 비율입니다.

<Info>
  ISMS-P에서 암호화가 요구되는 대상은 **민감 데이터를 포함한 리소스**입니다. 공개 정적 자산, 공개 문서 버킷 등 민감 데이터가 없는 리소스는 암호화 대상에서 제외될 수 있습니다. 따라서 전체 암호화율 수치보다 **미암호화 리소스에 민감 데이터가 포함되어 있는지**가 핵심 판단 기준입니다.
</Info>

**점검 방법:**

| 확인 항목 | 양호 | 미흡 |
| --- | --- | --- |
| 민감 데이터 포함 리소스가 모두 암호화되어 있는가 | 데이터 분류 기준에 따라 민감 리소스 100% 암호화 | 민감 데이터 포함 리소스 중 미암호화 존재 |
| 미암호화 리소스에 대한 사유가 문서화되어 있는가 | 공개 데이터 등 제외 사유 명시 | 사유 없이 미암호화 상태 방치 |
| 조직의 암호화 정책이 수립되어 있는가 | 데이터 분류 및 암호화 기준 정의됨 | 암호화 정책 부재 |

### S3 암호화 비율

S3 버킷 중 기본 암호화가 활성화된 비율입니다.

| 수치 | 판단 | 근거 |
| --- | --- | --- |
| 100% | 양호 | 모든 버킷에 기본 암호화 적용 |
| 100% 미만 | 확인 필요 | 미암호화 버킷의 데이터 분류 확인 필요 |

<Info>
  AWS는 2023년 1월부터 신규 버킷에 SSE-S3 기본 암호화를 자동 적용합니다. 따라서 최근 생성된 버킷은 기본적으로 암호화 상태입니다. 이전에 생성된 버킷 중 미암호화 상태인 경우, **해당 버킷에 민감 데이터가 포함되어 있는지** 확인하는 것이 우선입니다. 공개 정적 웹사이트 호스팅 버킷 등은 암호화 대상에서 제외할 수 있으나, 제외 사유를 문서화해야 합니다.
</Info>

**암호화 유형별 차이:**

| 유형 | 설명 | 비용 | 키 관리 |
| --- | --- | --- | --- |
| SSE-S3 | AWS 관리 키 (AES-256) | 무료 | AWS가 자동 관리 |
| SSE-KMS | KMS 고객 관리 키 | KMS API 호출 비용 | 직접 키 교체/정책 관리 |
| SSE-C | 고객 제공 키 | 무료 | 직접 키 보관/관리 필요 |

### EBS 암호화 비율

| 수치 | 판단 | 근거 |
| --- | --- | --- |
| 100% | 양호 | 모든 볼륨이 암호화됨 |
| 100% 미만 | 확인 필요 | 미암호화 볼륨의 용도 및 데이터 민감도 확인 필요 |

<Note>
  개발/테스트 환경의 볼륨에 민감 데이터가 없는 경우 암호화 대상에서 제외할 수 있습니다. 다만 제외 시 해당 볼륨이 민감 데이터를 포함하지 않음을 **데이터 분류 기준에 따라 확인하고 문서화**해야 합니다. 프로덕션 환경의 EBS 볼륨은 원칙적으로 모두 암호화해야 합니다.
</Note>

<Info>
  **조치:** 기존 미암호화 EBS는 직접 암호화할 수 없습니다. 스냅샷 생성 → 암호화 복사 → 새 볼륨 생성 → 교체 순서로 진행합니다. **예방:** 계정 수준에서 "EBS 암호화 기본값"을 활성화하면 신규 볼륨은 자동 암호화됩니다.
</Info>

### 암호화 실패 건수

암호화 관련 점검에서 FAIL인 총 건수입니다.

## 주요 점검 항목

| 점검 항목 | 설명 | 심각도 |
| --- | --- | --- |
| S3 버킷 기본 암호화 | 버킷 암호화 활성화 여부 | High |
| EBS 볼륨 암호화 | 볼륨 암호화 적용 여부 | Medium |
| RDS 스토리지 암호화 | RDS 암호화 여부 | High |
| KMS 키 자동 교체 | 키 자동 교체 활성화 | Medium |
| 로드밸런서 SSL/TLS | SSL/TLS 리스너 사용 여부 | High |
