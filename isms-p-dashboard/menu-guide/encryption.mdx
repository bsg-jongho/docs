<Warning>
  암호화는 데이터 유출 시 최후의 방어선입니다. S3 버킷 미암호화, EBS 볼륨 미암호화, 전송 중 암호화 미적용은 데이터가 평문으로 노출되는 결과를 초래합니다.
</Warning>

## 핵심 KPI

### 전체 암호화 준수율

| 수치 | 판단 | 조치 |
| --- | --- | --- |
| 95-100% | 양호 | 잔여 미암호화 리소스 확인 |
| 80-94% | 주의 | 미암호화 리소스 식별 후 암호화 적용 |
| 80% 미만 | 경고 | 조직 차원의 암호화 정책 수립 필요 |

### S3 암호화 비율

S3 버킷 중 기본 암호화가 활성화된 비율입니다.

| 수치 | 판단 | 근거 |
| --- | --- | --- |
| 100% | 양호 | 모든 버킷에 기본 암호화 적용 |
| 100% 미만 | 경고 | 미암호화 버킷의 모든 객체가 평문 상태 |

<Info>
  AWS는 2023년 1월부터 신규 버킷에 SSE-S3 기본 암호화를 자동 적용합니다. 이전에 생성된 버킷은 수동 설정이 필요합니다.
</Info>

**암호화 유형별 차이:**

| 유형 | 설명 | 비용 | 키 관리 |
| --- | --- | --- | --- |
| SSE-S3 | AWS 관리 키 (AES-256) | 무료 | AWS가 자동 관리 |
| SSE-KMS | KMS 고객 관리 키 | KMS API 호출 비용 | 직접 키 교체/정책 관리 |
| SSE-C | 고객 제공 키 | 무료 | 직접 키 보관/관리 필요 |

### EBS 암호화 비율

| 수치 | 판단 | 근거 |
| --- | --- | --- |
| 100% | 양호 | 모든 볼륨이 암호화됨 |
| 100% 미만 | 주의 | 미암호화 볼륨은 스냅샷 통한 데이터 유출 위험 |

<Info>
  **조치:** 기존 미암호화 EBS는 직접 암호화할 수 없습니다. 스냅샷 생성 → 암호화 복사 → 새 볼륨 생성 → 교체 순서로 진행합니다. **예방:** 계정 수준에서 "EBS 암호화 기본값"을 활성화하면 신규 볼륨은 자동 암호화됩니다.
</Info>

### 암호화 실패 건수

암호화 관련 점검에서 FAIL인 총 건수입니다.

## 주요 점검 항목

| 점검 항목 | 설명 | 심각도 |
| --- | --- | --- |
| S3 버킷 기본 암호화 | 버킷 암호화 활성화 여부 | High |
| EBS 볼륨 암호화 | 볼륨 암호화 적용 여부 | Medium |
| RDS 스토리지 암호화 | RDS 암호화 여부 | High |
| KMS 키 자동 교체 | 키 자동 교체 활성화 | Medium |
| 로드밸런서 SSL/TLS | SSL/TLS 리스너 사용 여부 | High |
