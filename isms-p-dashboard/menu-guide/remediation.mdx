<Info>
  단일 시점의 준수율만으로는 보안 관리의 효과를 판단할 수 없습니다. 개선 현황 시트는 **시간에 따른 보안 상태 변화 방향**을 보여주며, ISMS-P 인증 심사 시 "지속적인 보안 관리 활동"의 증빙 자료로 활용할 수 있습니다.
</Info>

## 변경 유형 상세

### NEW (신규 실패)

전일에는 존재하지 않았으나 오늘 새로 발견된 FAIL 항목입니다.

**발생 원인:**
- 새 리소스가 보안 기준을 충족하지 않은 상태로 생성됨
- 기존 리소스의 설정이 변경되어 FAIL로 전환됨
- 새 점검 규칙이 추가되어 기존 리소스가 FAIL로 판정됨

### FIXED (해결)

전일에 FAIL이었으나 오늘 PASS로 전환된 항목입니다.

- FIXED > NEW → 보안 상태 **개선 중** (긍정적)
- FIXED < NEW → 보안 상태 **악화 중** (조치 속도보다 새 이슈 발생이 빠름)
- FIXED = 0이 지속 → 보안 조치 활동이 없는 것. 담당자 확인 필요

### REGRESSION (재발)

<Warning>
  **가장 중요한 지표입니다.** 이전에 PASS였다가 다시 FAIL로 전환된 항목입니다.

  | REGRESSION 패턴 | 원인 추정 | 조치 |
  | --- | --- | --- |
  | 동일 항목 반복 재발 | 자동화가 설정을 덮어씀 | IaC 코드에 보안 설정 반영 |
  | 특정 계정에서만 재발 | 해당 계정 관리자가 설정 변경 | 계정 관리자와 협의, 정책 교육 |
  | 대량 일괄 재발 | 인프라 재배포 또는 복원 | 배포 파이프라인에 보안 점검 추가 |
</Warning>

## 추이 그래프 읽는 법

**건강한 추세:**
- FIXED가 NEW+REGRESSION보다 많은 상태 유지
- REGRESSION이 감소 추세
- 전체 FAIL 건수가 감소 추세

**위험 신호:**
- NEW가 지속적으로 FIXED보다 많음 → FAIL 누적 중
- REGRESSION이 증가 추세 → 조치의 지속성 문제
- FIXED가 0인 날이 계속됨 → 보안 조치 활동 중단

## 수동 점검 필요 항목

대시보드 하단에는 자동 스캔으로 판정할 수 없어 **담당자가 직접 확인해야 하는 ISMS-P 점검 항목** 목록이 별도로 표시됩니다.

이 항목들은 ISMS-P 인증 기준에는 포함되지만 AWS API만으로는 PASS/FAIL을 자동 판정할 수 없는 항목입니다. 예를 들어 보안 정책의 문서화 여부, 교육 실시 이력, 물리적 보안 조치 등이 해당됩니다.

<Info>
  수동 점검 항목은 자동 준수율 계산에 포함되지 않습니다. 그러나 ISMS-P 인증 심사에서는 이 항목들도 동일하게 평가되므로, 별도의 증빙 자료를 준비해야 합니다. 대시보드에 표시된 항목 목록을 체크리스트로 활용하세요.
</Info>
