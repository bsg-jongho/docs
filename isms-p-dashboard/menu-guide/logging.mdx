<Warning>
  **로깅은 다른 모든 보안 도메인의 기반 인프라입니다.** 로깅이 미설정되면 보안 사고를 탐지할 수 없고, 사후 분석도 불가능합니다. 다른 보안 도메인에 비해 로깅은 높은 준수율이 실질적으로 중요한 영역입니다.
</Warning>

## 다른 시트에 미치는 영향

로깅 시트의 FAIL은 다른 시트의 **데이터 신뢰성에 직접 영향**을 줍니다:

- CloudTrail 비활성 → CloudTrail 이벤트 수집이 불완전
- VPC Flow Logs 미활성 → 네트워크 보안 사각지대
- S3 접근 로깅 미활성 → 데이터 유출 여부 확인 불가

## 핵심 KPI

### 로깅 준수율

로깅은 보안 포렌식의 기반이므로 다른 도메인보다 높은 준수율이 중요합니다. 다만, 모든 환경에서 100%가 필수는 아닙니다. 개발/테스트 환경이나 대량 트래픽의 비민감 서비스는 비용 대비 효과를 고려하여 의도적으로 로깅을 제외할 수 있습니다.

| 상황 | 심각도 | 조치 |
| --- | --- | --- |
| 프로덕션 환경 핵심 서비스 로깅 완비 | - | 제외 항목에 대한 근거 문서화 확인 |
| 프로덕션 일부 서비스 로깅 누락 | High | 누락된 서비스 식별 후 중요도 순 설정 |
| 프로덕션 핵심 서비스(CloudTrail, VPC Flow Logs 등) 로깅 미설정 | Critical | 즉시 조치 필요. 사고 발생 시 원인 분석 불가 |

<Info>
  준수율 수치 자체보다 **"무엇이 로깅되지 않고 있는가"**가 중요합니다. 70%라도 프로덕션 핵심 시스템이 모두 커버되고, 나머지 30%가 개발 환경이라면 합리적일 수 있습니다. 반대로 95%라도 CloudTrail이 빠져 있다면 심각한 문제입니다.
</Info>

<Info>
  **ISMS-P 영향:** 로깅 미비는 다수 인증 기준에 동시에 영향을 줍니다.
  - 2.9.1 로그 및 접속기록 관리
  - 2.9.2 로그 및 접속기록 점검
  - 2.11.1 침해사고 대응 (로그 없이 대응 불가)
  - 2.11.3 사후관리 (로그 없이 원인 분석 불가)
</Info>

### 로깅 실패 건수

로깅 관련 점검에서 FAIL인 총 건수입니다.

## 주요 점검 항목

| 점검 항목 | 설명 | 심각도 | FAIL 시 영향 |
| --- | --- | --- | --- |
| CloudTrail 멀티리전 활성화 | 모든 리전 로깅 여부 | Critical | 특정 리전 API 호출 미기록 |
| 로그 파일 무결성 검증 | CloudTrail 무결성 검증 | High | 공격자의 로그 조작 탐지 불가 |
| S3 접근 로깅 | S3 데이터 접근 기록 | Medium | 데이터 접근 이력 추적 불가 |
| CloudWatch 로그 보존 기간 | 로그 보존 설정 여부 | Medium | 로그가 너무 빨리 삭제되거나 무기한 보존으로 비용 증가 |
| VPC Flow Logs | 네트워크 트래픽 기록 | High | 네트워크 감사 불가 |
