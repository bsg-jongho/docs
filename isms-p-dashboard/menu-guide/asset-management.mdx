특정 도메인에 국한되지 않고, **리소스 관점**에서 보안 상태를 파악하는 시트입니다. ISMS-P 인증 기준 중 **2.1 정보자산 관리** 영역에 해당합니다.

## 핵심 KPI

### 전체 리소스 수

대시보드가 점검한 고유 리소스의 총 수입니다.

<Info>
  이 수치가 갑자기 크게 변하면 (1) 새 계정이 추가되었거나 (2) 대량의 리소스가 생성/삭제된 것입니다. 변동 원인을 확인하세요.
</Info>

### 점검 통과율

| 수치 | 판단 | 조치 |
| --- | --- | --- |
| 90% 이상 | 양호 | FAIL 리소스 개별 확인 |
| 70-89% | 주의 | 서비스 그룹별 FAIL 집중 영역 파악 |
| 70% 미만 | 경고 | 전반적 보안 기준선 재수립 필요 |

### FAIL 리소스 수

하나 이상의 점검 항목에서 FAIL인 리소스의 수입니다.

### 전체 점검 규칙 수

적용된 보안 점검 규칙의 총 수입니다.

## 서비스 그룹별 분류

리소스를 서비스 그룹으로 분류하여 보여줍니다:

| 서비스 그룹 | 포함 리소스 |
| --- | --- |
| IAM | IAM 사용자, 역할, 정책, 액세스 키 |
| S3 | S3 버킷, 버킷 정책 |
| EC2/VPC | EC2 인스턴스, 보안 그룹, VPC, 서브넷 |
| Lambda | Lambda 함수, 레이어 |
| RDS | RDS 인스턴스, 클러스터 |
| KMS | KMS 키 |
| CloudTrail | CloudTrail 트레일 |
| 기타 | 위에 포함되지 않는 서비스 |

<Info>
  **활용 방법:** 특정 서비스 그룹의 FAIL 비율이 높다면, 해당 서비스의 보안 정책을 일괄 점검하세요. 예: S3 그룹의 FAIL이 많으면 버킷 정책, 암호화, 퍼블릭 접근 설정을 일괄 점검합니다.
</Info>
