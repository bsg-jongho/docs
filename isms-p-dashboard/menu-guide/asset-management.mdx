특정 도메인에 국한되지 않고, **리소스 관점**에서 보안 상태를 파악하는 시트입니다. ISMS-P 인증 기준 중 **2.1 정보자산 관리** 영역에 해당합니다.

## 핵심 KPI

### 전체 리소스 수

대시보드가 점검한 고유 리소스의 총 수입니다.

<Info>
  이 수치가 갑자기 크게 변하면 (1) 새 계정이 추가되었거나 (2) 대량의 리소스가 생성/삭제된 것입니다. 변동 원인을 확인하세요.
</Info>

### 점검 통과율

점검 통과율은 목표 수치를 정해놓고 달성하는 지표가 아닙니다. ISMS-P 인증에서 중요한 것은 FAIL 항목을 **인지하고 있는지**, 그리고 각 FAIL에 대해 **조치 또는 수용 근거가 문서화되어 있는지**입니다.

| 상황 | 판단 | 조치 |
| --- | --- | --- |
| FAIL 항목이 모두 식별되고 수용 근거 또는 조치 계획이 문서화됨 | 양호 | 정기적으로 수용 근거의 유효성 재검토 |
| FAIL 항목 중 미확인 또는 미문서화 건이 존재 | 주의 | 미확인 FAIL 항목을 서비스 그룹별로 분류하여 우선순위 지정 |
| FAIL 항목 다수가 관리되지 않고, 원인 파악 미완료 | 경고 | 서비스 그룹별 FAIL 집중 영역 파악 후 보안 기준선 재정립 |

<Warning>
  통과율 숫자에 집중하면 본질을 놓칩니다. 예를 들어 90%라도 FAIL인 10%가 프로덕션 IAM 정책이라면 심각한 문제이고, 60%라도 FAIL의 대부분이 개발 환경의 비핵심 리소스이며 수용 근거가 문서화되어 있다면 적절한 상태일 수 있습니다. **"무엇이 FAIL인가"와 "그에 대한 의사결정이 있는가"를 확인하세요.**
</Warning>

### FAIL 리소스 수

하나 이상의 점검 항목에서 FAIL인 리소스의 수입니다.

### 전체 점검 규칙 수

적용된 보안 점검 규칙의 총 수입니다.

## 서비스 그룹별 분류

리소스를 서비스 그룹으로 분류하여 보여줍니다:

| 서비스 그룹 | 포함 리소스 |
| --- | --- |
| IAM | IAM 사용자, 역할, 정책, 액세스 키 |
| S3 | S3 버킷, 버킷 정책 |
| EC2/VPC | EC2 인스턴스, 보안 그룹, VPC, 서브넷 |
| Lambda | Lambda 함수, 레이어 |
| RDS | RDS 인스턴스, 클러스터 |
| KMS | KMS 키 |
| CloudTrail | CloudTrail 트레일 |
| 기타 | 위에 포함되지 않는 서비스 |

<Info>
  **활용 방법:** 특정 서비스 그룹의 FAIL 비율이 높다면, 해당 서비스의 보안 정책을 일괄 점검하세요. 예: S3 그룹의 FAIL이 많으면 버킷 정책, 암호화, 퍼블릭 접근 설정을 일괄 점검합니다.
</Info>
