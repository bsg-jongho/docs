## 핵심 지표 읽는 법

각 지표가 의미하는 바와 수치별 대응 방법을 안내합니다.

### 전체 ISMS-P 준수율

대시보드에 등록된 모든 계정의 점검 항목 중 PASS인 비율입니다. 조직 전체의 보안 상태를 개괄적으로 파악하는 출발점이지만, **이 수치 자체가 보안 수준을 결정하지는 않습니다.**

<Info>
  **준수율보다 중요한 것:** ISMS-P 인증의 핵심은 "100% PASS"가 아니라, 조직이 자사의 보안 현황을 정확히 파악하고 있으며 미준수 항목에 대해 **합리적인 사유와 보완 통제가 있는지** 입증하는 것입니다. 예를 들어 웹 서버의 80/443 포트 개방, ALB의 0.0.0.0/0 인바운드 규칙은 서비스 운영상 정상적인 구성입니다.
</Info>

**이 수치를 활용하는 방법:**

1. **FAIL 항목의 내용을 확인하세요.** 준수율이 85%라도 FAIL 항목이 모두 문서화된 예외(서비스 특성상 불가피한 구성 등)라면 문제가 없을 수 있습니다.
2. **의도된 미준수인지 구분하세요.** 운영상 필요한 설정(공개 웹 서비스의 포트 개방 등)과 실제 보안 취약점(Root MFA 미설정 등)은 완전히 다릅니다.
3. **미인지 항목에 집중하세요.** 가장 위험한 것은 준수율 숫자가 아니라, 조직이 파악하지 못한 FAIL 항목입니다. 새로 발견된 FAIL이나 사유가 문서화되지 않은 항목을 우선 검토하세요.

<Warning>
  ISMS-P 인증 심사에서는 자동화 스캔 100% PASS가 아니라, **적절한 통제 수단의 적용**을 평가합니다. 다만 Critical 심각도 항목(Root MFA 미설정, 공개 S3 버킷 등)은 예외 사유가 인정되기 어려우므로 즉시 조치가 필요합니다. **준수율 수치보다 FAIL 항목의 심각도와 예외 사유 존재 여부를 우선 확인**하세요.
</Warning>

### FAIL 건수

전체 점검 항목 중 FAIL 상태인 항목의 총 수입니다. 이 수치 자체보다 **전일 대비 증감**과 **어떤 항목이 FAIL인지**가 중요합니다.

- 전일 대비 증가했다면 → 새로운 문제가 발생했거나 리소스가 추가된 것. 하단 상세 테이블에서 새로운 FAIL 항목 확인
- 전일 대비 감소했다면 → 조치가 진행되고 있는 것. [개선 현황](/isms-p-dashboard/menu-guide/remediation) 시트에서 FIXED 항목 확인
- FAIL 건수가 있더라도 모두 문서화된 예외 항목이라면 운영상 문제가 없을 수 있음

### Critical FAIL 건수

심각도가 Critical인 FAIL 항목의 수입니다. 즉시 악용 가능한 보안 취약점이므로, 다른 지표보다 먼저 확인해야 합니다.

| 상황 | 대응 |
| --- | --- |
| 0건 | 다른 도메인 시트에서 High 항목 확인으로 이동 |
| 1건 이상 | 각 항목이 실제 취약점인지 확인. 오탐이 아닌 경우 24시간 내 조치 |

<Info>
  Critical 건수가 있더라도 먼저 **개별 항목을 확인**하세요. 자동화 스캔의 오탐(false positive)이거나 보완 통제가 이미 적용된 경우가 있을 수 있습니다. 단, Root MFA 미설정, 공개 S3 버킷 등 명백한 취약점은 즉시 조치가 필요합니다.
</Info>

### 오늘 조치 완료 건수

금일 스캔에서 전일 대비 FAIL에서 PASS로 전환된(FIXED) 항목의 수입니다. 보안 개선 활동의 성과를 한눈에 확인할 수 있습니다.

- 이 수치가 꾸준히 유지되면 보안 조치 활동이 정상적으로 진행되고 있는 것
- 0이 지속되면 보안 조치 활동이 중단된 것일 수 있으므로 담당자 확인 필요
- 변경의 상세 내역(NEW, FIXED, REGRESSION 분류)은 [개선 현황](/isms-p-dashboard/menu-guide/remediation) 시트에서 확인할 수 있음

### 도메인별 준수율 비교

각 도메인(접근권한, 네트워크, 암호화 등)의 준수율을 시각적으로 비교합니다. 가장 낮은 도메인부터 우선 조치하세요.
