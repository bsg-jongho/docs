## KPI 읽는 법

대시보드의 각 시트 상단에 표시되는 **핵심 성과 지표(KPI)**입니다. 각 보안 도메인의 현재 상태를 숫자 하나로 요약합니다.

### KPI 유형

**비율(%) KPI** — 예: Root MFA 비율, S3 암호화 비율, VPC Flow Logs 활성화율
- 산출: (PASS 건수 / 전체 건수) x 100, 목표값: **100%**

| 범위 | 상태 |
| --- | --- |
| 100% | 완전 준수 |
| 95-99% | 소수 미준수 |
| 80-94% | 개선 필요 |
| 80% 미만 | 긴급 조치 필요 |

**건수(Count) KPI** — 예: Critical 실패 건수, 위험 보안 그룹 수, 미보호 리소스 수
- 목표값: **0**

**추세(Trend) KPI** — 예: 일일 변경 추이 (NEW, FIXED, REGRESSION)
- 목표값: **FIXED > NEW + REGRESSION**

<Warning>
  비율이 높아도 안심할 수 없습니다. 95%라도 나머지 5%가 Critical 항목이면 위험합니다. **비율과 함께 심각도를 반드시 확인**하세요.
</Warning>

### 심각도(Severity) 기준

| 심각도 | ISMS-P 영향 | 대응 시간 |
| --- | --- | --- |
| **Critical** | 인증 불가 수준의 결함 | 즉시 (24시간 내) |
| **High** | 인증 심사 시 지적 가능 | 1주 내 |
| **Medium** | 권고 사항 | 1개월 내 |
| **Low** | 개선 권고 | 분기 내 |
