## 개요

대시보드는 최대 20개의 AWS 계정을 동시에 모니터링할 수 있습니다. 계정을 추가하면 대상 계정에 읽기 전용 스캔 역할(`ISMS-P-ScanRole`)이 **자동으로 생성**되며, 이후 매일 스캔이 수행됩니다.

## 계정 추가

### CLI를 통한 추가

```
isms-p target add <계정 ID>
```

실행하면 대상 계정의 IAM 자격 증명(Access Key, Secret Key)을 입력하라는 안내가 표시됩니다.

| 입력 항목 | 설명 |
| --- | --- |
| 계정 ID | 추가할 AWS 계정의 12자리 ID |
| Access Key ID | 대상 계정에서 생성한 IAM 사용자의 액세스 키 |
| Secret Access Key | 해당 액세스 키의 시크릿 키 |

<Info>
  대상 계정의 IAM 사용자에는 **IAMFullAccess** 권한이 필요합니다. 이 자격 증명은 스캔 역할을 자동 생성하는 데만 사용되며, 역할 생성이 완료된 후에는 해당 IAM 사용자와 액세스 키를 삭제해도 됩니다.
</Info>

### 자동으로 일어나는 일

자격 증명을 입력하면 이후 과정은 모두 자동으로 처리됩니다.

| 순서 | 처리 내용 |
| --- | --- |
| 1 | 입력한 자격 증명이 안전하게 저장됨 |
| 2 | Terraform이 대상 계정 목록을 업데이트 |
| 3 | 다음 스캔 시작 시, 대상 계정에 `ISMS-P-ScanRole`이 자동 생성됨 |
| 4 | 생성된 역할로 읽기 전용 보안 데이터 수집 시작 |

<Info>
  **ScanRole은 읽기 전용입니다.** 대상 계정의 리소스를 수정하거나 삭제하지 않습니다. 부여되는 권한은 `ViewOnlyAccess`(리소스 조회)와 `SecurityAudit`(보안 감사)뿐입니다.
</Info>

### 보안 장치

대상 계정에 생성되는 스캔 역할에는 다음과 같은 보안 장치가 적용됩니다.

| 보안 장치 | 설명 |
| --- | --- |
| ExternalId | 중앙 계정 ID 기반의 고유 식별자. 다른 AWS 계정이 이 역할을 무단으로 사용하는 것을 방지 |
| 읽기 전용 정책 | ViewOnlyAccess + SecurityAudit만 부여. 리소스 변경 권한 없음 |
| 세션 제한 | 최대 1시간의 세션 유효 기간 |

## 계정 제거

```
isms-p target remove <계정 ID>
```

제거 시 대상 계정의 `ISMS-P-ScanRole`도 자동으로 삭제됩니다. 이미 수집된 과거 데이터는 보존됩니다 (원본 데이터 90일, 분석 데이터 영구).

## 병렬 처리와 스캔 시간

최대 5개 계정이 동시에 스캔되며, 계정 수에 따라 전체 소요 시간이 달라집니다.

| 계정 수 | 예상 스캔 시간 |
| --- | --- |
| 1-5 | 30-60분 |
| 6-10 | 60-90분 |
| 11-20 | 90-120분 |
