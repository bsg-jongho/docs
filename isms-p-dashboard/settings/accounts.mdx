## 개요

대시보드는 여러 AWS 계정을 동시에 모니터링할 수 있습니다. 스캔 대상 계정은 S3에 저장된 설정 파일(`config/accounts.json`)로 관리됩니다.

## 계정 추가

새 AWS 계정을 스캔 대상에 추가하려면 2단계가 필요합니다.

### 1단계: 대상 계정에 ScanRole 생성

대상 계정에 `ISMS-P-ScanRole`을 생성해야 합니다. 이 역할은 대시보드(중앙 계정)가 대상 계정의 데이터를 읽기 위해 사용합니다.

| 흐름 | 설명 |
| --- | --- |
| 중앙 계정 (대시보드) | Lambda 실행 역할이 AssumeRole + ExternalId로 대상 계정 접근 |
| → 대상 계정 ScanRole | 읽기 전용으로 보안 데이터 수집 |

**필요 권한:** 보안 감사를 위한 읽기 권한 (SecurityAudit, ViewOnlyAccess), Security Hub 읽기, CloudTrail 읽기

<Info>
  **ExternalId란?** 혼동된 대리인(Confused Deputy) 공격을 방지하기 위한 보안 장치입니다. 중앙 계정 ID가 포함되어 있어, 다른 AWS 계정이 이 역할을 무단으로 위임하는 것을 방지합니다.
</Info>

<Tip>
  **자동 프로비저닝:** `ensure_scan_roles` 기능이 활성화되어 있으면, ScanRole이 자동으로 생성/업데이트됩니다.
</Tip>

### 2단계: Terraform 변수에 계정 추가

Terraform 변수 `target_account_ids`에 새 계정 ID를 추가하고 적용하면, `accounts.json`이 자동으로 업데이트됩니다.

## 계정 제거

1. Terraform 변수 `target_account_ids`에서 해당 계정 ID 제거
2. Terraform 적용
3. (선택) 대상 계정에서 `ISMS-P-ScanRole` 삭제

<Info>
  계정을 제거해도 이미 수집된 과거 데이터는 S3에 보존됩니다 (raw/ 90일, processed/ 영구).
</Info>

## 병렬 처리 제한

| 계정 수 | 예상 스캔 시간 |
| --- | --- |
| 1-5 | 30-60분 |
| 6-10 | 60-90분 |
| 11-20 | 90-120분 |
